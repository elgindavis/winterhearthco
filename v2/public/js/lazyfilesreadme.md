    <article class="markdown-body entry-content p-3 p-md-6" itemprop="text"><h1><a id="user-content-lazysizes" class="anchor" aria-hidden="true" href="#lazysizes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>lazysizes</h1>
<p><strong>lazysizes</strong> is a fast (jank-free), SEO-friendly and self-initializing lazyloader for images (including responsive images <code>picture</code>/<code>srcset</code>), iframes, scripts/widgets and much more. It also prioritizes resources by differentiating between crucial in view and near view elements to make perceived performance even faster.</p>
<p>It may become also your number one tool to integrate responsive images. It can automatically calculate the <code>sizes</code> attribute for your responsive images, it allows you to share media queries for your <code>media</code> attributes with your CSS, helping to separate layout (CSS) from content/structure (HTML) and it makes integrating responsive images into any environment really simple. It also includes a set of optional plugins to further extend its functionality.</p>
<h2><a id="user-content-how-to" class="anchor" aria-hidden="true" href="#how-to"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>How to</h2>

<h2><a id="user-content-responsive-image-support-picture-andor-srcset" class="anchor" aria-hidden="true" href="#responsive-image-support-picture-andor-srcset"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Responsive image support (picture and/or srcset)</h2>
<p>Lazysizes is built upon the Responsive image standard and extends it with additional functionality. For full cross browser responsive image support you must use either a full polyfill like <a href="https://github.com/scottjehl/picturefill">picturefill</a> or use the extreme lightweight partial <a href="/aFarkas/lazysizes/blob/gh-pages/plugins/respimg">respimg polyfill plugin</a> or the <a href="/aFarkas/lazysizes/blob/gh-pages/plugins/rias">responsive image on demand plugin</a>. Alternatively, you can simply define a fallback src via the <code>data-src</code> attribute. If you want to learn more about the responsive image syntax read "<a href="https://jakearchibald.com/2015/anatomy-of-responsive-images/" rel="nofollow">The anatomy of responsive images</a>".</p>
<h2><a id="user-content-what-makes-lazysizes-so-awesome" class="anchor" aria-hidden="true" href="#what-makes-lazysizes-so-awesome"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>What makes lazysizes so awesome:</h2>
<p><strong>lazysizes</strong> is different than other lazy image loaders.</p>
<ol>
<li><strong>Detects any visibility changes on current and future lazyload elements in any web environment automatically</strong>: The script works as an universal, self-initializing, self-configuring and self-destroying component and detects any changes to the visibility of any current and future image/iframe elements automatically no matter whether it becomes visible through a user scroll, a CSS animation triggered through <code>:hover</code> or through any kind of JS behavior (carousel, slider, infinite scroll, masonry, isotope/filtering/sorting, AJAX, SPAs...). It also works automatically in conjunction with any kind of JS-/CSS-/Frontend-Framework (jQuery mobile, Bootstrap, Backbone, Angular, React, Ember (see also the <a href="/aFarkas/lazysizes/blob/gh-pages/plugins/attrchange">attrchange/re-initialization extension</a>)).</li>
<li><strong>Future-proof</strong>: It directly includes standard responsive image support (<code>picture</code> and <code>srcset</code>)</li>
<li><strong>Separation of concerns</strong>: For responsive image support it adds an automatic <code>sizes</code> calculation as also alias names for media queries feature. There is also no JS change needed if you add a scrollable container with CSS (overflow: auto) or create a mega menu containing images.</li>
<li><strong>Performance</strong>: It's based on highly efficient, best practice code (runtime <strong>and</strong> network) to work jank-free at 60fps and can be used with hundreds of images/iframes on CSS and JS-heavy pages or webapps.</li>
<li><strong>Extendable</strong>: It provides JS and CSS hooks to extend lazysizes with any kind of lazy loading, lazy instantiation, in view callbacks or effects (see also the <a href="#plugins">available plugins/snippets</a>).</li>
<li><strong>Intelligent prefetch/Intelligent resource prioritization</strong>: lazysizes prefetches/preloads near the view assets to improve user experience, but only while the browser network is idling (see also <code>expand</code>, <code>expFactor</code> and <code>loadMode</code> options). This way in view elements are loaded faster and near of view images are preloaded lazily before they come into view.</li>
<li><strong>Lightweight, but mature solution</strong>: lazysizes has the right balance between a lightweight and a fast, reliable solution</li>
<li><strong>SEO improved</strong>: lazysizes does not hide images/assets from Google. No matter what markup pattern you use. Google doesn't scroll/interact with your website. lazysizes detects, whether the user agent is capable to scroll and if not, reveals all images instantly.</li>
</ol>
<h2><a id="user-content-more-about-the-api" class="anchor" aria-hidden="true" href="#more-about-the-api"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>More about the API</h2>
<p><strong>lazysizes</strong> comes with a simple markup and JS API. Normally you will only need to use the markup API.</p>
<h3><a id="user-content-markup-api" class="anchor" aria-hidden="true" href="#markup-api"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Markup API</h3>
<p>Add the <code>class</code> <code>lazyload</code> to all <code>img</code> and <code>iframe</code> elements, which should be loaded lazy. <em>Instead</em> of a <code>src</code> or <code>srcset</code> attribute use a <code>data-src</code> or <code>data-srcset</code> attribute:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">img</span> <span class="pl-c1">data-src</span>="<span class="pl-s">image.jpg</span>" <span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;
<span class="pl-c">&lt;!-- retina optimized image: --&gt;</span>
<span class="pl-kos">&lt;</span><span class="pl-ent">img</span> <span class="pl-c1">data-srcset</span>="<span class="pl-s">responsive-image1.jpg 1x, responsive-image2.jpg 2x</span>" <span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;</pre></div>
<h4><a id="user-content-automatically-setting-the-sizes-attribute" class="anchor" aria-hidden="true" href="#automatically-setting-the-sizes-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><a name="user-content-data-sizes-auto"></a>Automatically setting the <code>sizes</code> attribute</h4>
<p><strong>lazysizes</strong> supports setting the <code>sizes</code> attribute automatically, corresponding to the current size of your image - just set the value of <code>data-sizes</code> to  <code>auto</code>.</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">img</span>
	<span class="pl-c1">data-sizes</span>="<span class="pl-s">auto</span>"
	<span class="pl-c1">data-srcset</span>="<span class="pl-s">responsive-image1.jpg 300w,</span>
<span class="pl-s">	    responsive-image2.jpg 600w,</span>
<span class="pl-s">	    responsive-image3.jpg 900w</span>"
    <span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;</pre></div>
<p><strong><a name="user-content-sizes-calculation"></a>Important: How <code>sizes</code> is calculated</strong>: The automatic sizes calculation uses the display width of the image. This means that the width of the image has to be calculable at least approximately before the image itself is loaded (This means you can not use <code>width: auto</code>). Often the following general CSS rule might help: <code>img[data-sizes="auto"] { display: block; width: 100%; }</code> (see also <a href="#specify-dimensions">specifying image/iframe dimensions with the recommended aspect ratio definition</a>). If it is below <code>40</code> (can be configured through the <code>minSize</code> option), lazysizes traverses up the DOM tree until it finds a parent which is over <code>40</code> and uses this number.</p>
<p>The width auto-calculated by lazysizes can be modified using the <code>lazybeforesizes</code> event (<a href="#lazybeforesizes-documentation">lazybeforesizes documentation</a>). Alternatively, the <a href="/aFarkas/lazysizes/blob/gh-pages/plugins/parent-fit">parent fit plugin</a> can be used for sizing images to fit a parent / container, and is the only solution when an image's height needs to be taken into account when fitting it to its container (This also includes the use of <code>object-fit</code>).</p>
<p>The <code>data-sizes="auto"</code> feature only makes sense if you use the <code>data-srcset</code> attribute with <em>width</em> descriptors which allows the most appropriate image can be selected (It does not make sense if you use the x descriptor or only <code>src</code>.).</p>
<h2><a id="user-content-recommendedpossible-markup-patterns" class="anchor" aria-hidden="true" href="#recommendedpossible-markup-patterns"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Recommended/possible markup patterns</h2>
<p>lazysizes allows you to write an endless variety of different markup patterns. Find your own/best pattern or choose one of the following. (All of the following patterns can be also used for art direction using the <code>picture</code> element.)</p>
<h3><a id="user-content-simple-pattern" class="anchor" aria-hidden="true" href="#simple-pattern"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Simple pattern</h3>
<p>Add the class <code>lazyload</code> and simply omit the <code>src</code> attribute  or add a data uri as fallback <code>src</code>.</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-c">&lt;!--  responsive adaptive example --&gt;</span>

<span class="pl-kos">&lt;</span><span class="pl-ent">img</span>
	<span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>"
	<span class="pl-c1">data-srcset</span>="<span class="pl-s">image.jpg 1x, image2.jpg 2x</span>"
    <span class="pl-c1">alt</span>="<span class="pl-s">my image</span>" /&gt;
<span class="pl-c">&lt;!--  retina optimized example --&gt;</span>
<span class="pl-kos">&lt;</span><span class="pl-ent">img</span> <span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>"
	<span class="pl-c1">data-srcset</span>="<span class="pl-s">progressive-image.jpg 1x, progressive-image2.jpg 2x</span>"
    <span class="pl-c1">alt</span>="<span class="pl-s">my image</span>" /&gt;

<span class="pl-c">&lt;!-- or non-responsive: --&gt;</span>
<span class="pl-kos">&lt;</span><span class="pl-ent">img</span>
	<span class="pl-c1">data-src</span>="<span class="pl-s">image.jpg</span>"
	<span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;</pre></div>
<p>Note: In case you are using either <code>srcset</code>/<code>data-srcset</code> or <code>picture</code>, we recommend to extend this pattern with either a <code>data-src</code> (see next pattern: "Combine <code>data-srcset</code> with <code>data-src</code>") or with a suitable <code>src</code> attribute (see:  "modern pattern" or "LQIP").</p>
<h3><a id="user-content-combine-data-srcset-with-data-src" class="anchor" aria-hidden="true" href="#combine-data-srcset-with-data-src"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Combine <code>data-srcset</code> with <code>data-src</code></h3>
<p>In case you want to use responsive images for supporting browsers, but don't want to include a polyfill, simply combine your <code>data-srcset</code> with a <code>data-src</code> attribute.</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-c">&lt;!-- responsive example: --&gt;</span>
<span class="pl-kos">&lt;</span><span class="pl-ent">img</span>
	<span class="pl-c1">data-sizes</span>="<span class="pl-s">auto</span>"
    <span class="pl-c1">data-src</span>="<span class="pl-s">image3.jpg</span>"
	<span class="pl-c1">data-srcset</span>="<span class="pl-s">image3.jpg 600w,</span>
<span class="pl-s">	    image1.jpg 220w,</span>
<span class="pl-s">	    image2.jpg 300w,</span>
<span class="pl-s">	    image3.jpg 600w,</span>
<span class="pl-s">	    image4.jpg 900w</span>"
	<span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;</pre></div>
<p>Note: Due to the fact that the <code>data-src</code> will also be picked up by "Read-Later" Apps and other tools (for example Pin it button), this pattern also makes sense if you use a polyfill. In case you don't use a polyfill it is recommended that the first image candidate matches the fallback <code>src</code>.</p>
<h3><a id="user-content-lqipblurry-image-placeholderblur-up-image-technique" class="anchor" aria-hidden="true" href="#lqipblurry-image-placeholderblur-up-image-technique"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>LQIP/blurry image placeholder/Blur up image technique</h3>
<p>If you are using the LQIP (Low Quality Image Placeholder) pattern, simply add a low quality image as the <code>src</code>:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-c">&lt;!-- responsive example: --&gt;</span>
<span class="pl-kos">&lt;</span><span class="pl-ent">img</span>

    <span class="pl-c1">src</span>="<span class="pl-s">lqip-src.jpg</span>"
	<span class="pl-c1">data-srcset</span>="<span class="pl-s">lqip-src.jpg 220w,</span>

<span class="pl-s">    image3.jpg 600w,</span>

<span class="pl-kos">&lt;</span><span class="pl-ent">img</span> <span class="pl-c1">src</span>="<span class="pl-s">lqip-src.jpg</span>" <span class="pl-c1">data-src</span>="<span class="pl-s">image.jpg</span>" <span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;</pre></div>

<h3><a id="user-content-modern-transparent-srcset-pattern" class="anchor" aria-hidden="true" href="#modern-transparent-srcset-pattern"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>modern transparent <code>srcset</code> pattern</h3>
<p>Combine a normal <code>src</code> attribute with a transparent or low quality image as <code>srcset</code> value and a <code>data-srcset</code> attribute. This way modern browsers will lazy load without loading the <code>src</code> attribute and all others will simply fallback to the initial <code>src</code> attribute (without lazyload). (This nice pattern originated from @ivopetkov.)</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">img</span>
    <span class="pl-c1">src</span>="<span class="pl-s">image3.jpg</span>"
    <span class="pl-c1">srcset</span>="<span class="pl-s">data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==</span>"
	<span class="pl-c1">data-srcset</span>="<span class="pl-s">image3.jpg 600w,</span>
<span class="pl-s">		image1.jpg 220w,</span>
<span class="pl-s">	    image2.jpg 300w,</span>
<span class="pl-s">	    image4.jpg 900w</span>"
	<span class="pl-c1">data-sizes</span>="<span class="pl-s">auto</span>"
	<span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;</pre></div>
<h3><a id="user-content-the-noscript-pattern" class="anchor" aria-hidden="true" href="#the-noscript-pattern"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>The noscript pattern</h3>
<p>In case disabled JavaScript is a concern you can combine this simple pattern with an image inside a <code>noscript</code> element.</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">style</span><span class="pl-kos">&gt;</span>
	.<span class="pl-c1">no-js</span> <span class="pl-ent">img</span>.<span class="pl-c1">lazyload</span> {
    	<span class="pl-c1">display</span>: none;
    }
<span class="pl-kos">&lt;/</span><span class="pl-ent">style</span><span class="pl-kos">&gt;</span>

<span class="pl-c">&lt;!-- noscript pattern --&gt;</span>
<span class="pl-kos">&lt;</span><span class="pl-ent">noscript</span><span class="pl-kos">&gt;</span>
	<span class="pl-kos">&lt;</span><span class="pl-ent">img</span> <span class="pl-c1">src</span>="<span class="pl-s">image.jpg</span>" /&gt;
<span class="pl-kos">&lt;/</span><span class="pl-ent">noscript</span><span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;</span><span class="pl-ent">img</span> <span class="pl-c1">src</span>="<span class="pl-s">transparent.jpg</span>" <span class="pl-c1">data-src</span>="<span class="pl-s">image.jpg</span>" <span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;</pre></div>
<p>Note: As an alternative to the noscript pattern also checkout the <a href="/aFarkas/lazysizes/blob/gh-pages/plugins/noscript">noscript extension</a>.</p>
<h3><a id="user-content-data-expand-attribute" class="anchor" aria-hidden="true" href="#data-expand-attribute"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>[data-expand] attribute</h3>
<p>Normally lazysizes will expand the viewport area to lazy preload images/iframes which might become visible soon. This value can be adjusted using the <code>expand</code> option.</p>
<p>Additionally, this general option can be overridden with the <code>data-expand</code> attribute for each element. Different than the general <code>expand</code> option the <code>data-expand</code> attribute also accepts negative values (All numbers but <code>0</code> are accepted!).</p>
<p>This becomes especially handy to add unveiling effects for teasers or other elements:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">style</span><span class="pl-kos">&gt;</span>
.<span class="pl-c1">lazyload</span><span class="pl-kos">,</span>
.<span class="pl-c1">lazyloading</span> {
	<span class="pl-c1">opacity</span>: <span class="pl-c1">0</span>;
}
.<span class="pl-c1">lazyloaded</span> {
	<span class="pl-c1">opacity</span>: <span class="pl-c1">1</span>;
	<span class="pl-c1">transition</span>: opacity <span class="pl-c1">300<span class="pl-smi">ms</span></span>;
}
<span class="pl-kos">&lt;/</span><span class="pl-ent">style</span><span class="pl-kos">&gt;</span>

<span class="pl-kos">&lt;</span><span class="pl-ent">div</span> <span class="pl-c1">class</span>="<span class="pl-s">teaser lazyload</span>" <span class="pl-c1">data-expand</span>="<span class="pl-s">-20</span>"<span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">img</span> <span class="pl-c1">data-src</span>="<span class="pl-s">image.jpg</span>" <span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;
    <span class="pl-kos">&lt;</span><span class="pl-ent">h1</span><span class="pl-kos">&gt;</span>Teaser Title<span class="pl-kos">&lt;/</span><span class="pl-ent">h1</span><span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">p</span><span class="pl-kos">&gt;</span>...<span class="pl-kos">&lt;/</span><span class="pl-ent">p</span><span class="pl-kos">&gt;</span>
<span class="pl-kos">&lt;/</span><span class="pl-ent">div</span><span class="pl-kos">&gt;</span></pre></div>
<h3><a id="user-content-css-api" class="anchor" aria-hidden="true" href="#css-api"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>CSS API</h3>

<h3><a id="user-content-broken-image-symbol" class="anchor" aria-hidden="true" href="#broken-image-symbol"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Broken image symbol</h3>
<p>In case you are using an <code>alt</code> attribute but do not declare a <code>src</code>/<code>srcset</code> attribute you will end up with a broken image symbol.</p>
<p>There are two easy ways to deal with it.</p>
<p>Either define a <code>src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw=="</code> or add the following CSS.</p>
<div class="highlight highlight-source-css"><pre><span class="pl-ent">img</span>.<span class="pl-c1">lazyload</span>:<span class="pl-c1">not</span>([<span class="pl-c1">src</span>]) {
	<span class="pl-c1">visibility</span>: hidden;

<p><strong>lazysizes</strong> automatically detects new elements with the class <code>lazyload</code> so you won't need to call or configure anything in most situations.</p>
<h4><a id="user-content-js-api---options" class="anchor" aria-hidden="true" href="#js-api---options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>JS API - options</h4>
<p>Options can be set by declaring a global configuration option object named <code>lazySizesConfig</code>. This object must be defined before the lazysizes script. A basic example:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-smi">window</span><span class="pl-kos">.</span><span class="pl-c1">lazySizesConfig</span> <span class="pl-c1">=</span> <span class="pl-smi">window</span><span class="pl-kos">.</span><span class="pl-c1">lazySizesConfig</span> <span class="pl-c1">||</span> <span class="pl-kos">{</span><span class="pl-kos">}</span><span class="pl-kos">;</span>

<div class="highlight highlight-source-js"><pre><span class="pl-k">import</span> <span class="pl-s1">lazySizes</span> <span class="pl-k">from</span> <span class="pl-s">'lazysizes'</span><span class="pl-kos">;</span>

<p>Here the list of options:</p>
<ul>
<li><code>lazySizesConfig.lazyClass</code> (default: <code>"lazyload"</code>): Marker class for all elements which should be lazy loaded (There can be only one <code>class</code>. In case you need to add some other element, without the defined class, simply add it per JS: <code>$('.lazy-others').addClass('lazyload');</code>)</li>
<li><code>lazySizesConfig.preloadAfterLoad</code> (default: <code>false</code>): Whether lazysizes should load all elements after the window onload event. Note: lazySizes will then still download those not-in-view images inside of a lazy queue, so that other downloads after onload are not blocked.)</li>
<li><code>lazySizesConfig.preloadClass</code> (default: <code>"lazypreload"</code>): Marker class for elements which should be lazy pre-loaded after onload. Those elements will be even preloaded, if the <code>preloadAfterLoad</code> option is set to <code>false</code>. Note: This <em>class</em> can be also dynamically set (<code>$currentSlide.next().find('.lazyload').addClass('lazypreload');</code>).</li>
<li><code>lazySizesConfig.loadingClass</code> (default: <code>"lazyloading"</code>): This <code>class</code> will be added to <code>img</code> element as soon as image loading starts. Can be used to add unveil effects.</li>
<li><code>lazySizesConfig.loadedClass</code> (default: <code>"lazyloaded"</code>): This <code>class</code> will be added to any element as soon as the image is loaded or the image comes into view. Can be used to add unveil effects or to apply styles.</li>
<li><code>lazySizesConfig.expand</code> (default: <code>370-500</code>): The <code>expand</code> option expands the calculated visual viewport area in all directions, so that elements can be loaded before they become visible. The default value is calculated depending on the viewport size of the device. (Note: Reasonable values are between <code>300</code> and <code>1000</code> (depending on the <code>expFactor</code> option.) In case you have a lot of small images or you are using the LQIP pattern you can lower the value, in case you have larger images set it to a higher value. Also note, that lazySizes will dynamically shrink this value to <code>0</code> if the browser is currently downloading and expand it if the browser network is currently idling and the user not scrolling (by multiplying the <code>expand</code> option with <code>1.5</code> (<code>expFactor</code>)). This option can also be overridden with the <code>[data-expand]</code> attribute.</li>
<li><code>lazySizesConfig.minSize</code> (default: <code>40</code>): For <code>data-sizes="auto"</code> feature. The minimum size of an image that is used to calculate the <code>sizes</code> attribute. In case it is under <code>minSize</code> the script traverses up the DOM tree until it finds a parent that is over <code>minSize</code>.</li>
<li><code>lazySizesConfig.srcAttr</code> (default: <code>"data-src"</code>): The attribute, which should be transformed to <code>src</code>.</li>
<li><code>lazySizesConfig.srcsetAttr</code> (default: <code>"data-srcset"</code>): The attribute, which should be transformed to <code>srcset</code>.</li>
<li><code>lazySizesConfig.sizesAttr</code> (default: <code>"data-sizes"</code>): The attribute, which should be transformed to <code>sizes</code>. Makes almost only makes sense with the value <code>"auto"</code>. Otherwise, the <code>sizes</code> attribute should be used directly.</li>
<li><code>lazySizesConfig.customMedia</code> (default: <code>{}</code>): The <code>customMedia</code> option object is an alias map for different media queries. It can be used to separate/centralize your multiple specific media queries implementation (layout) from the <code>source[media]</code> attribute (content/structure) by creating labeled media queries. (See also the <a href="/aFarkas/lazysizes/blob/gh-pages/plugins/custommedia">custommedia extension</a>).</li>
<li><code>lazySizesConfig.loadHidden</code> (default: <code>true</code>): Whether to load <code>visibility: hidden</code> elements. Important: lazySizes will load hidden images always delayed. If you want them to be loaded as fast as possible you can use <code>opacity: 0.001</code> but never <code>visibility: hidden</code> or <code>opacity: 0</code>.</li>
<li><code>lazySizesConfig.ricTimeout</code> (default: <code>0</code>): The timeout option used for the <code>requestIdleCallback</code>. Reasonable values between: 0, 100 - 1000. (Values below 50 disable the <code>requestIdleCallback</code> feature.)</li>
<li><code>lazySizesConfig.throttleDelay</code> (default: <code>125</code>): The timeout option used to throttle all listeners. Reasonable values between: 66 - 200.</li>
</ul>

<ul>
<li><code>lazySizesConfig.expFactor</code> (default: <code>1.5</code>): The <code>expFactor</code> is used to calculate the "preload expand", by multiplying the normal <code>expand</code> with the <code>expFactor</code> which is used to preload assets while the browser is idling (no important network traffic and no scrolling). (Reasonable values are between <code>1.5</code> and <code>4</code> depending on the <code>expand</code> option).</li>
<li><code>lazySizesConfig.hFac</code> (default: <code>0.8</code>): The <code>hFac</code> (horizontal factor) modifies the horizontal expand by multiplying the <code>expand</code> value with the <code>hFac</code> value. Use case: In case of carousels there is often the wish to make the horizontal expand narrower than the normal vertical expand option. Reasonable values are between 0.4 - 1. In the unlikely case of a horizontal scrolling website also 1 - 1.5.</li>
<li><code>lazySizesConfig.loadMode</code> (default: <code>2</code>): The <code>loadMode</code> can be used to constrain the allowed loading mode. Possible values are 0 = don't load anything, 1 = only load visible elements, 2 = load also very near view elements (<code>expand</code> option) and 3 = load also not so near view elements (<code>expand</code> * <code>expFactor</code> option). This value is automatically set to <code>3</code> after onload. Change this value to <code>1</code> if you (also) optimize for the onload event or change it to <code>3</code> if your onload event is already heavily delayed.</li>
<li><code>lazySizesConfig.init</code> (default: <code>true</code>): By default lazysizes initializes itself, to load in view assets as soon as possible. In the unlikely case you need to setup/configure something with a later script you can set this option to <code>false</code> and call <code>lazySizes.init();</code> later explicitly.</li>
</ul>

<li>
<p><a id="user-content-lazybeforesizes-documentation"></a><code>lazybeforesizes</code>: This event will be fired on each element with the <code>data-sizes="auto"</code> attribute right before the calculated <code>sizes</code> attribute will be set. In case the event is <code>defaultPrevented</code> the <code>sizes</code> attribute won't be set. See also the <a href="/aFarkas/lazysizes/blob/gh-pages/plugins/parent-fit">parent-fit extension</a>.</p>
</li>

<p>The <code>srcset</code> attribute with the <em>w</em> descriptor and <code>sizes</code> attribute automatically also includes high DPI images. But each image has a different optimal pixel density, which might be lower (for example 1.5x) than the pixel density of your device (2x or 3x). This information is unknown to the browser and therefore can't be optimized for. The <a href="/aFarkas/lazysizes/blob/gh-pages/plugins/optimumx">lazySizes optimumx extension</a> gives you more control to trade between perceived quality vs. perceived performance.</p>

<h2><a id="user-content-tip-specifying-image-dimensions-minimizing-reflows-and-avoiding-page-jumps" class="anchor" aria-hidden="true" href="#tip-specifying-image-dimensions-minimizing-reflows-and-avoiding-page-jumps"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><a name="user-content-specify-dimensions"></a>Tip: Specifying image dimensions (minimizing reflows and avoiding page jumps)</h2>
<p>To minimize reflows, content jumping or unpredictable behavior with some other JS widgets (isotope, masonry, some sliders/carousels...) the width <strong>and</strong> the height of an image should be calculable by the browser before the image source itself is loaded:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">img</span>

    <span class="pl-c1">style</span>="<span class="pl-s">width: 350px; height: 150px;</span>"
	<span class="pl-c1">data-srcset</span>="<span class="pl-s">http://placehold.it/350x150 1x,</span>
<span class="pl-s">    http://placehold.it/700x300 2x</span>"
    <span class="pl-c1">data-src</span>="<span class="pl-s">http://placehold.it/350x150</span>"
    <span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;</pre></div>
<p>For flexible responsive images the <a href="http://www.mademyday.de/css-height-equals-width-with-pure-css.html" rel="nofollow">CSS intrinsic ratio scaling technique</a> should be used:</p>
<div class="highlight highlight-text-html-basic"><pre><span class="pl-kos">&lt;</span><span class="pl-ent">style</span><span class="pl-kos">&gt;</span>
.<span class="pl-c1">ratio-container</span> {
    <span class="pl-c1">position</span>: relative;
}
.<span class="pl-c1">ratio-container</span>:<span class="pl-c1">after</span> {
    <span class="pl-c1">content</span>: <span class="pl-s">''</span>;
    <span class="pl-c1">display</span>: block;
    <span class="pl-c1">height</span>: <span class="pl-c1">0</span>;
    <span class="pl-c1">width</span>: <span class="pl-c1">100<span class="pl-smi">%</span></span>;
    <span class="pl-c">/* 16:9 = 56.25% = calc(9 / 16 * 100%) */</span>
    <span class="pl-c1">padding-bottom</span>: <span class="pl-c1">42.86<span class="pl-smi">%</span></span>;
}
.<span class="pl-c1">ratio-container</span> <span class="pl-c1">&gt;</span> <span class="pl-ent"><span class="pl-c1">*</span></span> {
    <span class="pl-c1">position</span>: absolute;
    <span class="pl-c1">top</span>: <span class="pl-c1">0</span>;
    <span class="pl-c1">left</span>: <span class="pl-c1">0</span>;
    <span class="pl-c1">width</span>: <span class="pl-c1">100<span class="pl-smi">%</span></span>;
    <span class="pl-c1">height</span>: <span class="pl-c1">100<span class="pl-smi">%</span></span>;
    <span class="pl-c1">display</span>: block;
}
<span class="pl-kos">&lt;/</span><span class="pl-ent">style</span><span class="pl-kos">&gt;</span>

<span class="pl-kos">&lt;</span><span class="pl-ent">div</span> <span class="pl-c1">class</span>="<span class="pl-s">ratio-container</span>"<span class="pl-kos">&gt;</span>
    <span class="pl-kos">&lt;</span><span class="pl-ent">img</span>

        <span class="pl-c1">data-sizes</span>="<span class="pl-s">auto</span>"
        <span class="pl-c1">data-srcset</span>="<span class="pl-s">http://placehold.it/175x75 175w,</span>
<span class="pl-s">        http://placehold.it/350x150 350w,</span>
<span class="pl-s">        http://placehold.it/700x300 700w,</span>
<span class="pl-s">        http://placehold.it/1400x600 1400w</span>"

        <span class="pl-c1">class</span>="<span class="pl-s">lazyload</span>" /&gt;
<span class="pl-kos">&lt;/</span><span class="pl-ent">div</span><span class="pl-kos">&gt;</span></pre></div>

<p>In case you can't specify the image dimensions using CSS or one of the above suggested methods and your JS widgets have problems to calculate the right dimensions. You can use the following pattern to update your JS widgets (sliders/masonry):</p>

        <span class="pl-s1">$module</span><span class="pl-kos">.</span><span class="pl-en">myWidget</span><span class="pl-kos">(</span><span class="pl-s">'updateLayout'</span><span class="pl-kos">)</span><span class="pl-kos">;</span>

<h2><a id="user-content-tip-wherehow-to-include-lazysizes" class="anchor" aria-hidden="true" href="#tip-wherehow-to-include-lazysizes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><a id="user-content-include-early"></a>Tip: Where/How to include lazySizes</h2>
<p>While lazy loading is a great feature, it is important for users that crucial in view images are loaded as fast as possible. (Most users start to interact with a page after in view images are loaded.)</p>
<p>In case you normally combine all your scripts into one large script and add this to the bottom of your page, it can be better for perceived performance to generate two or sometimes more script packages: One small package, which includes all scripts which have heavy influence on the content or the UI and another larger one which includes the normal behavior of the page.</p>
<p>This smaller script, which should include lazySizes (and all its plugins), should than be placed <strong>before</strong> any other blocking elements (i.e.: script(s)) at the end of the body or after any blocking elements (i.e.: scripts, stylesheets) in the head to load the crucial content as fast possible. (Note: It might make also sense to call <code>lazySizes.init();</code> explicitly right after lazySizes and all its plugins are added.)</p>
<h2><a id="user-content-why-lazysizes" class="anchor" aria-hidden="true" href="#why-lazysizes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Why lazysizes</h2>
<p>In the past, I often struggled using lazy image loaders, because the "main check function" is called repeatedly and with a high frequency. Which makes it hard to fulfill two purposes runtime and memory efficiency. And looking into the source code of most so called lazy loaders often also unveils lazy developers...</p>
<p>But in a world of responsive retina optimized images on the one hand and JS widgets like carousels or tabs (a lot of initially hidden images) on the other hand lazy loading images becomes more and more important, so I created this project.</p>
<p><strong>lazysizes</strong> is different:</p>
<p>Due to the fact, that it is designed to be invoked with a high frequency and therefore works highly efficient, it was possible to hook into all kinds of events as a mutationobserver meaning this lazyloader works as a simple drop in solution - you simply write/render your markup and no matter whether the <code>.lazyload</code> element was added by AJAX or revealed by a JS or CSS animation it will be picked up by <strong>lazysizes</strong>.</p>
